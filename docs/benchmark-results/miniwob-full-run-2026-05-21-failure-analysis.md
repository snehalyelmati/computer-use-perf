# MiniWoB++ Full Suite All-Cases Review - 2026-05-21

This report audits every episode from the one-repeat MiniWoB++ full-suite run for Zip. BrowserGym refers to this benchmark with the key `miniwob`.

The retained study path is `2026-05-21_22-05-01_computer-use-agent-on-miniwob-full`; it includes the earlier `computer-use-agent` working name because the run predates the Zip rename. The report uses retained AgentLab artifacts as source of truth: `per_task_results.csv`, `result_df.csv`, `summary_info.json`, `goal_object.pkl.gz`, and each `step_*.pkl.gz`. Native `logs/agentlab/<run_id>/` files were checked only when still present.

The audit was cross-checked through 25 five-case assignments run in five-agent waves, then consolidated with a local artifact/source pass for the long-running social-media and terminal/editor assignments. Scratch extraction files remain outside the repository under `/tmp/miniwob_failure_review/` and `/tmp/miniwob_subagent_audit/`.

## Executive Summary

- The full suite contains `125` completed episodes: `73` reward-positive successes and `52` zero-reward failures. The failure split is `20` truncated at `env_max_steps=10` and `32` zero-reward non-truncated failures. Errors are `0`.
- Reported AgentLab stats show failures dominating cost and tokens: `$35.6537` and `16,734,835` tokens versus `$3.1471` and `1,568,871` tokens for successes. These stats are useful for concentration analysis, but multi-step episodes may overstate true native run cost because retained AgentLab rows sum cumulative internal counters.
- Email inbox tasks remain the largest failure family: `10/10` failed and all truncated. In each case the agent reached useful email state, then lost the decisive CSS-image icon control such as Send, Forward, Star, or Trash.
- The success path is narrow but reliable: `62` of `73` successes completed in one BrowserGym step, mostly on native buttons, inputs, links, options, autocomplete suggestions, and clearly represented widgets.
- Highest-impact fixes are generic exposure of handler-bearing icon spans/divs, pointer/widget tools for drag/draw/focus/slider/resize, non-interactive visual/text/table context, and reward-aware recovery after negative BrowserGym validation.

## Representative Deep Dives

- Email inbox: `miniwob.email-inbox` seed `15` opened Noemi's email, clicked Forward, typed `Simona`, then looped looking for Send. The HTML uses CSS-image `span#send-forward`; similar send, star, and trash spans explain all ten inbox failures. Fix area: handler-bearing icon spans in `src/agent/context/snapshot.py` and `src/agent/context/handlers.py`.
- Drag/draw/widgets: `drag-items-grid`, `resize-textarea`, `use-slider`, and `draw-line` show the same theme from different widgets: targets were either not exposed as draggable/resize/slider surfaces or the semantic tool used DOM drag/clicks where pointer gestures or set-value semantics were required. Fix area: `src/agent/tools/semantic.py` plus richer widget geometry in `snapshot.py`.
- Click/visual: `circle-center`, `find-midpoint`, and `right-angle` calculated plausible SVG coordinates, but BrowserGym rejected them. The evidence points to coordinate-frame ambiguity and lack of reward-aware stop after bad submits, not just model math. Fix area: `click_at` coordinate contract and SVG geometry summaries.
- Form/text/content: `text-transform` missed transformed non-interactive captcha text, `read-table` selected the wrong row value, `multi-orderings` confused input labels after row order changed, and `focus-text` used DOM click where a focus event was required. Fix area: non-interactive text/label structure plus a real focus action.
- Social media: all three failures targeted posts and containers, not the empty CSS-image `.reply`, `.retweet`, or `.like` spans. The target user often existed offscreen, so the issue is action-control exposure with parent post context, not absence of data.
- Terminal: `terminal` seed `21` repeatedly issued `ls`/wait patterns and never deleted the extensionless file `image`. The agent needed a durable terminal-output readback path and loop aborts after unchanged output.

## Reviewability Findings

- The AgentLab study artifacts were sufficient to audit all `125` episodes because every episode retained `summary_info.json`, `goal_object.pkl.gz`, and `step_*.pkl.gz`.
- Native runtime logs were retained for only `10` episodes because `src/agent/metrics.py` prunes old UUID run directories. This is an observability issue, not direct evidence of task failure.
- `benchmarks/agentlab/computer_use_agent.py` returning `noop()` is expected: semantic tools mutate the BrowserGym page, then BrowserGym validates the page state.
- Cost/token fields in this report are the retained AgentLab per-task stats. For multi-step episodes, subagent audit found examples where those fields sum cumulative internal counters and exceed retained native `run_summary.json` totals. Keep them as a relative concentration signal unless native run summaries are retained for every episode.
- Persisting the latest BrowserGym reward/status, per-tool arguments/results, and a compact native trace inside each `AgentInfo.extra_info` would make future reviews less dependent on pruned UUID log directories.
